# 🚀 Weather Alert System – Admin Portal

A minimalist, invitation-based weather alert system built using NestJS and React. The application allows public users to request access, while administrators can review requests, approve users, and monitor real-time weather alerts generated by a background scheduler.

This project demonstrates clean monorepo architecture, role-based access control, reactive frontend state management, and a pragmatic startup-ready approach to full-stack development.

---

# 📊 Project Overview

The objective of this project is to build an interactive administrative dashboard to manage user access and dispatch automated weather alerts based on user approval status.

The application supports:
* Public invite requests
* Secure Admin login and authorization
* User approval/rejection workflows
* Automated background weather alerts via Cron jobs
* Reactive UI with optimistic updates

# 🔬 Project Workflow & Architecture

## 1. 🏗️ Clean Monorepo Layout
To improve maintainability and deployment efficiency, the application is structured as a monorepo containing two distinct layers:
* **Frontend:** Manages the user interface, routing, and server state polling.
* **Backend:** A modular monolith using NestJS. Controllers remain thin, routing requests directly to highly-focused Services.

## 2. 🗄️ Pragmatic Database Decision (SQLite vs PostgreSQL)
While PostgreSQL is a standard choice for enterprise applications, **SQLite was selected for this assessment**. This intentional choice ensures rapid, zero-friction local setup for reviewers without requiring Docker or external services. This perfectly aligns with the requested "startup-ready" mindset of focusing on functionality, speed, and developer experience first.

## 3. 🎨 UI Design & Styling
A clean, premium glassmorphism dark-themed interface was designed to improve usability and visual consistency.
* Custom Tailwind utility components
* Responsive layout
* Micro-animations and interactive hover states

## 4. ⏰ Automated Weather Scheduler
The backend utilizes `@nestjs/schedule` to run a Cron job.
* Checks the database for users with the `APPROVED` status
* Generates randomized mock weather alerts
* Persists alerts to the database for real-time frontend consumption

# ▶️ Running the Project Locally

## 1. Backend Setup

Navigate to the backend directory:
```bash
cd backend
```

Install dependencies:
```bash
npm install
```

Generate the Prisma client and run the seed script (this creates the default Admin user):
```bash
npx prisma generate
npx tsx prisma/seed.ts
```

Run the server:
```bash
npm run start:dev
```
*Backend runs on `http://localhost:3000`*

*(The seed script automatically creates the admin user: `tony.AI47Labs@gmail.com` / `tony@47`)*

---

## 2. Frontend Setup

Open a new terminal and navigate to the frontend directory:
```bash
cd frontend
```

Install dependencies:
```bash
npm install
```

Start the application:
```bash
npm run dev
```
*Frontend runs on `http://localhost:5173`*
